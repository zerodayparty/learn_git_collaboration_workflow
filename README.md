# 🟩 실전 Git 협업 워크플로우  

이 프로젝트는 3~5인 팀이 함께 협업하며 실전 Git과 GitHub 흐름을 배우는 실습 프로젝트 가이드임. 코드를 개발하는 기술보다, 브랜치 관리, Pull Request(PR), 코드 리뷰, 충돌(Conflict) 해결, 트러블슈팅 등 협업의 규칙과 문화를 몸으로 학습하는 것을 목적으로 함.  

<br><br>

## 🟢 프로젝트 디렉토리 구조  

| 폴더 및 파일 | 설명 |  
| --- | --- |
| `_practice/` | 실습을 단계별로 진행할 수 있도록 설계된 교육 가이드 문서들이 위치함. |  
| `_learned/` | 학습자가 공부하면서 새롭게 배운 내용들을 정리하여 기록하는 공간임. |  
| `_record/` | 매 작업이 완료될 때마다 작업의 세부 내용을 일지 형태로 기록하는 공간임. |  
| `_temporary/` | 실습 과정에서 생성되는 임시 파일들을 저장하는 공간임. |  
| `docs/` | 협업 규칙(`CONTRIBUTING.md`), 충돌 해결 기록(`conflict-resolution.md`), 트러블슈팅 기록(`troubleshooting-log.md`) 등 팀 공통 문서가 관리됨. |  
| `src/` | 실습을 위해 작성하는 소스 코드나 관련 파일들이 저장되는 폴더임. |  

<br><br>

## 🟢 실습 가이드 목차 (Roadmap)  

학습을 진행하기 위해 `_practice/` 폴더 안의 가이드 문서를 20번부터 짝수 번호 순서대로 학습해야 함.  

- [20_과제_전체_흐름_요약]  
    - 실습의 전체 요구사항 및 전체적인 진행 순서를 요약함.  
- [22_팀_저장소_준비]  
    - GitHub Organization 또는 개인 저장소를 구성하고 main 브랜치를 보호(Branch Protection)하는 방법을 설명함.  
- [24_이슈_브랜치_PR_흐름]  
    - GitHub Flow 브랜치 전략과 이슈 생성, 브랜치 생성, PR(Pull Request) 등록의 전체적 흐름을 다룸.  
- [26_CONTRIBUTING_협업규칙_작성]  
    - 팀원들이 협업 시 준수해야 할 브랜치 명명법, 커밋 컨벤션 등을 `CONTRIBUTING.md`에 작성하는 법을 학습함.  
- [28_팀원별_결과물_기여]  
    - 팀원들이 각자 브랜치를 통해 간단한 유틸 함수나 자기소개, 학습 노트를 저장소에 올리는 실습을 수행함.  
- [30_PR_리뷰_승인_병합]  
    - 단순 승인이 아닌 구체적 라인 피드백 중심의 코드 리뷰를 진행하고 PR을 main에 병합하는 방법을 다룸.  
- [32_충돌_해결_실습]  
    - 동일 파일 수정 등을 통해 비자명 충돌 상황을 의도적으로 유발하고, 마커를 확인하며 이를 해결하는 실습임.  
- [34_Git_트러블슈팅_실습]  
    - `commit --amend`, `reset --soft`, `revert`, `stash` 4가지 실수 복구 및 작업 전환 시나리오를 팀원별로 실습함.  
- [36_SUBMISSION_제출_증빙]  
    - 완성된 실습 결과를 모아 제출하기 위해 `SUBMISSION.md` 인덱스 문서를 정리하는 법을 다룸.  
- [38_보너스_히스토리_CODEOWNERS]  
    - 인터랙티브 리베이스(`rebase -i`)를 통한 히스토리 정리 및 `CODEOWNERS`를 활용한 리뷰어 자동화 보너스 미션임.

## 🟢 Git 명령어 및 협업 용어 사전  

초보자가 명령어와 용어를 쉽게 이해할 수 있도록 풀네임과 해설을 정리한 표임.  

### 🟡 Git 핵심 명령어  

| 명령어 | 풀네임 (Full Name) | 쉬운 설명 및 해설 |  
| --- | --- | --- |
| `git init` | initialize (초기화) | 현재 디렉토리를 Git이 관리하는 저장소로 설정함. |  
| `git status` | status (상태) | 어떤 파일이 수정되었고 스테이징 영역에 올라갔는지 상태를 확인함. |  
| `git add` | add (추가) | 수정된 파일들을 기록하기 전 준비 영역(Staging Area)에 올려둠. |  
| `git commit` | commit (확약/기록) | 준비 영역에 올린 파일들의 변경 내역을 메시지와 함께 로컬에 영구히 저장함. |  
| `git diff` | difference (차이) | 파일의 어떤 줄이 추가되었고 삭제되었는지 상세한 차이점을 보여줌. |  
| `git log` | log (기록) | 지금까지 커밋된 역사(히스토리)들을 시간 순서대로 조회함. |  
| `git push` | push (밀어넣기) | 로컬 저장소에 쌓인 커밋 기록들을 GitHub 원격 저장소로 전송함. |  
| `git pull` | pull (당겨오기) | 원격 저장소의 최신 커밋을 로컬 저장소로 가져와 즉시 병합(merge)함. |  
| `git checkout` | checkout (체크아웃/확인) | 특정 브랜치로 이동하거나 특정 과거 커밋 시점으로 파일을 전환함. |  
| `git branch` | branch (가지/분기) | 독립적인 작업 공간(가지)을 만들고, 확인하고, 삭제하는 도구임. |  
| `git merge` | merge (병합) | 다른 브랜치에서 작업한 내용을 현재 브랜치로 가져와 하나로 합침. |  
| `git reset` | reset (재설정) | 과거의 커밋 시점으로 돌아가며 이후 기록을 삭제하거나 준비 영역을 초기화함. |  
| `git revert` | revert (되돌리기) | 과거 커밋을 취소하는 새로운 커밋을 추가하여 기존 기록을 훼손하지 않고 복구함. |  
| `git stash` | stash (임시저장/숨기기) | 하던 작업을 커밋하지 않고 임시 저장소에 안전하게 보관하여 브랜치를 전환할 수 있게 함. |

### 🟡 GitHub 협업 용어  

| 용어 | 풀네임 (Full Name) | 쉬운 설명 및 해설 |  
| --- | --- | --- |
| **CLI** | Command Line Interface (명령줄 인터페이스) | 터미널 검은 화면에 글자로 명령어를 직접 입력하여 컴퓨터를 제어하는 방식임. |  
| **PR** | Pull Request (풀 리퀘스트 / 당김 요청) | 내가 feature 브랜치에서 작업한 내용을 main 브랜치에 합쳐달라고 팀원들에게 요청하는 것임. |  
| **URL** | Uniform Resource Locator (자원 위치 지정자) | 인터넷 상의 특정 파일이나 페이지의 주소(예: 저장소 링크)임. |  
| **Repository** | Repository (저장소) | Git이 프로젝트 소스 코드와 버전 히스토리를 관리하는 폴더 또는 공간임. |  
| **Branch** | Branch (나뭇가지 / 분기) | main 브랜치의 코드를 복사하여 서로 다른 작업을 독립적으로 진행할 수 있게 만드는 길임. |  
| **Commit** | Commit (기록 / 커밋) | 의미 있는 변경 사항의 상태를 스냅샷처럼 찍어서 영구 저장하는 행위 또는 저장된 결과물임. |  
| **Conflict** | Conflict (충돌) | 같은 파일의 같은 라인을 서로 다르게 수정하여 머지할 때 Git이 자동으로 판단하지 못하고 멈추는 상황임. |

## 🟢 최종 미션 성공 기준 (통과 조건)  

- **팀 저장소 운영**: `main` 브랜치에 직접 push가 불가능하도록 Branch Protection Rule 설정 완료.  
- **개별 기여**: 팀원 전원이 각각 PR 최소 2개 이상 생성 및 병합 성공.  
- **코드 리뷰**: 팀원 전원이 타인의 PR에 구체적 피드백을 담은 리뷰 2개 이상 작성.  
- **충돌 해결**: 비자명 충돌 1회를 포함하여 총 2회 이상 충돌을 직접 해결하고 `docs/conflict-resolution.md`에 기록.  
- **트러블슈팅**: `commit --amend`, `reset --soft`, `revert`, `stash` 4가지 문제 해결 절차를 밟고 `docs/troubleshooting-log.md`에 기록.  
- **인덱스 문서**: `SUBMISSION.md`에 모든 팀원의 PR 링크 및 실습 결과 문서 링크 취합 완료.
