# 🟩 git merge 명령어  
## 🟢 명령어 개요  
### 🟡 git merge란 무엇인가  
갈라져 나와 독립적으로 작업을 진행했던 다른 브랜치(가지)의 변경 사항들을 현재 내가 머물고 있는 브랜치로 가져와 하나로 합치는 명령어이다.  
- `git` = Global Information Tracker (분산 버전 관리 시스템)  
- `merge` = merge (병합하다, 합치다)  
- 해설: 여러 사람이 나누어 개발한 각각의 기능들을 메인 브랜치(main)에 최종 통합할 때 사용한다.  

<br><br>

## 🟢 주요 사용법과 단어 풀네임  

| 명령어 | 명령어 영어 풀네임 및 단어 해석 | 해설 |
| :--- | :--- | :--- |
| `git merge [대상브랜치]` | `git` = Global Information Tracker<br>`merge` = merge (합치다) | 현재 내가 체크아웃(위치)해 있는 브랜치로 `[대상브랜치]`의 수정 내역을 가져와 결합한다. |
| `git merge --abort` | `--abort` = abort (중단하다, 취소하다) | 병합 도중 충돌(Conflict)이 발생했을 때, 병합 작업을 즉시 취소하고 병합 시작 직전 상태로 파일을 되돌린다. |

<br><br>

## 🟢 대표적인 병합 방식의 분류  

### 🟡 1. Fast-Forward (빨리 감기) 병합  
- `Fast-Forward` = fast forward (테이프/비디오를 빨리 감다)  
- **상황**: 브랜치가 갈라져 나간 이후, 메인 브랜치에는 아무런 새로운 변경 사항이 생기지 않았을 때 발생한다.  
- **방식**: Git은 별도로 코드를 합치는 연산을 하지 않고, 메인 브랜치가 가리키는 시점을 단순히 다른 브랜치의 최신 커밋 위치로 "빨리 감기"하듯 쑥 옮기기만 하고 종료한다.  
- **특징**: 병합 결과물로 새로운 커밋(Merge Commit)이 생성되지 않는다.  

### 🟡 2. 3-Way (3자 비교) 병합  
- `3-Way` = three way (3자 간, 세 방향)  
- **상황**: 브랜치가 갈라져 나간 이후, 메인 브랜치와 다른 브랜치 양쪽 모두에 새로운 변경 사항이 각각 누적되었을 때 발생한다.  
- **방식**: Git이 ① 두 브랜치가 최초로 갈라진 공통 조상 커밋, ② 메인 브랜치의 최신 커밋, ③ 다른 브랜치의 최신 커밋 이렇게 총 세 개의 지점(3-Way)을 서로 비교하여 자동으로 소스코드를 합쳐준다.  
- **특징**: 병합이 완료되면 "두 브랜치를 하나로 합쳤다"는 사실을 증명하는 새로운 역사적 커밋인 **병합 커밋(Merge Commit)**이 반드시 생성된다.  

<br><br>

## 🟢 직관적인 작동 비유  
`git merge`는 게임에서 **서브 퀘스트 세이브 파일을 메인 세이브 파일에 덮어씌워 통합하는 것**과 같다.  
- 메인 세이브 슬롯(main 브랜치)에 서브 퀘스트 슬롯(새 브랜치)의 게임 달성 정보(코드 수정 사항)를 합침으로써, 최종적으로 모든 진행 정보가 누적된 최신 상태의 세이브 파일 하나를 만들어 내는 행위이다.
