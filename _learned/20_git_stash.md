# 🟩 작업 임시 보관하기 (git stash)  

코드를 열심히 수정하고 있었는데 갑자기 다른 급한 업무를 처리하거나 브랜치를 변경해야 할 때, 작업 중인 불완전한 코드를 임시로 서랍에 넣어 보관하는 `git stash`에 대해 배운다.  



<br><br>

## 🟢 git stash 란?  
- 'Stash'는 '숨겨두다, 안전한 곳에 넣어두다'라는 뜻이다.  
- 아직 커밋하지 않은 작업 중인 변경 사항(Working Directory 및 Staging Area에 있는 상태)을 임시 저장소에 안전하게 저장해 두고, 내 작업 공간을 깨끗한 상태(직전 커밋 상태)로 비워준다.  



<br><br>

## 🟢 언제 사용하나요?  
- 브랜치 A에서 기능 개발 중인데, 급한 버그가 생겨 브랜치 B로 전환해야 할 때. (커밋되지 않은 수정 사항이 있으면 브랜치 전환이 막히거나 꼬인다.)  
- 작업물이 완성되지 않아 커밋 메시지를 남겨 버전으로 만들기는 찝찝하고, 그렇다고 변경 사항을 지울 수도 없을 때 사용한다.  



<br><br>

## 🟢 stash 기본 명령어  

### 🟡 작업 임시 보관하기  
- 현재 수정한 모든 변경 사항을 숨긴다.  
    ```bash
    git stash  
    ```
- 숨기면서 어떤 작업인지 메모를 남기려면 아래 명령어를 사용한다. (권장)  
    ```bash
    git stash -m "로그인 기능 개발 중 임시 저장"  
    ```

### 🟡 임시 보관 목록 확인하기  
- 보관된 작업 리스트를 확인한다. 가장 최근 보관한 작업이 `stash@{0}`번이 된다.  
    ```bash
    git stash list  
    ```

### 🟡 보관한 작업 꺼내와서 적용하기  
- 숨겨두었던 작업을 꺼내와서 현재 작업 트리에 다시 합친다.  
- **꺼내면서 서랍에서 삭제하기** (가장 많이 씀)  
    ```bash
    git stash pop  
    ```
- **꺼내오되 서랍에 그대로 남겨두기**  
    ```bash
    git stash apply  
    ```

### 🟡 보관한 작업 지우기  
- 서랍에 있는 임시 작업 내역을 삭제한다.  
- **특정 작업 지우기** (예: `stash@{0}`번 삭제)  
    ```bash
    git stash drop stash@{0}  
    ```
- **서랍 전체 비우기**  
    ```bash
    git stash clear  
    ```

## 🟢 실습: 작업 도중 브랜치 전환하기  

### 🟡 1. 파일 수정하기  
- `main` 브랜치에서 `stash_test.txt` 파일을 만들고 커밋한 뒤, 파일 내용을 임의로 수정한다. (아직 커밋하지 않은 상태 유지)  
    ```bash
    echo "원본 내용" > stash_test.txt  
    git add stash_test.txt  
    git commit -m "docs: add stash_test.txt"  
    echo "수정 중인 내용 (임시)" >> stash_test.txt  
    ```

### 🟡 2. 임시 보관하기  
- 수정 사항을 보관하여 작업 공간을 비운다.  
    ```bash
    git stash -m "임시 저장 실습"  
    ```
- `git status`를 쳐보면 작업 공간이 깨끗하다고 나온다.  

### 🟡 3. 안전하게 다른 브랜치 다녀오기  
- 작업 공간이 비워졌으므로 이제 안전하게 다른 브랜치로 전환할 수 있다.  
    ```bash
    git switch -c feature/temp-branch  
    # 다른 작업 수행...  
    git switch main  
    ```

### 🟡 4. 보관한 작업 다시 가져오기  
- `main` 브랜치로 돌아와서 아까 작업하던 내용을 서랍에서 꺼낸다.  
    ```bash
    git stash pop  
    ```
- `stash_test.txt` 파일을 열어보면 "수정 중인 내용 (임시)"가 다시 복구되어 있는 것을 확인할 수 있다.
