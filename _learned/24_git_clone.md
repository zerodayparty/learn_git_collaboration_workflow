# 🟩 git clone 명령어  

## 🟢 명령어 개요  

### 🟡 git clone이란 무엇인가  
인터넷 상(GitHub 등)에 이미 존재하는 원격 저장소(Remote Repository)의 모든 파일과 전체 버전 기록(History)을 그대로 복제하여 내 컴퓨터에 새 저장소로 내려받는 명령어이다.  
- `git` = Global Information Tracker (분산 버전 관리 시스템)  
- `clone` = clone (복제하다, 클론)  
- 해설: 새로운 개발 환경을 세팅하거나, 다른 사람의 오픈소스 프로젝트를 내 컴퓨터로 가져와서 개발을 시작할 때 최초 한 번 사용한다.  



<br><br>

## 🟢 주요 사용법과 단어 풀네임  

| 명령어 | 명령어 영어 풀네임 및 단어 해석 | 해설 |  
| :--- | :--- | :--- |
| `git clone [원격 저장소 URL]` | `clone` = clone (복제하다) | 입력한 원격 저장소의 주소에서 모든 소스 코드와 버전 기록을 다운로드하고, 폴더 내에 `.git` 폴더를 자동으로 구성하여 Git 저장소로 만든다. |  
| `git clone [원격 저장소 URL] [폴더명]` | `[폴더명]` = 다운로드할 디렉터리 이름 | 복제본을 다운로드할 때 기본 프로젝트 이름 대신 내가 지정한 `[폴더명]`의 새로운 폴더를 생성하여 그 안에 다운로드한다. |  



<br><br>

## 🟢 git init과 git clone의 결정적 차이  
두 명령어 모두 내 컴퓨터에 Git 저장소(.git)를 생성하는 명령어지만 실행 환경과 대상이 다르다.  

| 비교 항목 | git init | git clone |  
| :--- | :--- | :--- |
| **명령어 뜻** | `init` = initialize (초기화하다) | `clone` = clone (복제하다) |  
| **사용 대상** | 내 로컬 컴퓨터의 빈 폴더 | 이미 GitHub 등에 올라와 있는 원격 저장소 |  
| **작동 결과** | 아무 기록도 없는 최초의 빈 로컬 저장소를 만든다. | 원격 저장소에 쌓여 있던 수십, 수백 개의 기존 커밋 기록까지 통째로 복사해 온다. |  
| **원격 연결** | `git remote add` 명령을 통해 수동으로 원격지를 연결해 주어야 한다. | 복제해 오면서 `origin`이라는 이름으로 원격 저장소 주소를 자동으로 연결해 준다. |  



<br><br>

## 🟢 직관적인 작동 비유  
`git clone`은 문서 보관소(GitHub)에 보관된 **두꺼운 서류철 전체를 복사기로 복사(Clone)해서 내 책상 위로 가져오는 행위**와 같다.  
- 서류철 안에는 첫 페이지부터 마지막 페이지까지의 문서 변천사(Commit History)가 모두 들어있다.  
- 단순히 현재 상태의 최신 파일들만 복사해 오는 것이 아니라, 과거 수정 내역 전체를 그대로 담고 있는 복제본을 만드는 것이다.
