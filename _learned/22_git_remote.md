# 🟩 git remote 명령어  

## 🟢 명령어 개요  

### 🟡 git remote란 무엇인가  
내 컴퓨터(Local)의 Git 저장소와 인터넷 서버(GitHub, GitLab 등)에 있는 원격 저장소(Remote Repository)를 연결하고 관리하는 명령어이다.  
- `git` = Global Information Tracker (분산 버전 관리 시스템)  
- `remote` = remote (원격의, 멀리 떨어진)  
- 해설: 작성한 코드 버전을 클라우드 저장소에 올리거나 팀원의 코드를 내려받기 위해 원격 주소를 등록하는 역할을 한다.  

<br><br>

## 🟢 주요 사용법과 단어 풀네임  

| 명령어 | 명령어 영어 풀네임 및 단어 해석 | 해설 |  
| :--- | :--- | :--- |
| `git remote add origin [URL]` | `remote` = remote (원격의)<br>`add` = add (추가하다)<br>`origin` = origin (기원/근원, 기본 별칭)<br>`URL` = Uniform Resource Locator (인터넷 주소) | 지정한 인터넷 주소([URL])를 `origin`이라는 별칭의 원격 저장소로 내 로컬 프로젝트에 추가 등록한다. |  
| `git remote -v` | `-v` = verbose (상세한, 말이 많은) | 현재 등록되어 있는 원격 저장소의 단축 이름(별칭)과 실제 인터넷 주소(URL)를 상세히 보여준다. |  
| `git remote remove [이름]` | `remove` = remove (제거하다) | 등록되어 있는 특정 원격 저장소(예: origin)의 연결을 해제한다. (인터넷 상의 실제 저장소가 지워지는 것이 아니라 내 컴퓨터와의 연결만 끊어진다.) |  
| `git remote set-url [이름] [새URL]` | `set-url` = set URL (인터넷 주소를 설정하다) | 기존에 등록된 원격 저장소의 인터넷 주소를 새로운 주소로 변경한다. |  

<br><br>

## 🟢 'origin' 단어의 의미 비밀  
- Git에서 원격 저장소를 연결할 때 왜 관습적으로 `origin`이라는 이름을 사용할까?  
- `origin`은 '기원, 근원'이라는 뜻의 영어 단어이다.  
- 내 컴퓨터 저장소의 근원이 되는 인터넷 상의 모태 저장소를 의미하며, Git이 자동으로 부여하는 가장 표준적인 기본 별칭일 뿐이므로 사용자가 원한다면 다른 단어(예: server, main_db)로 이름을 바꿔 등록해도 무방하다.  

<br><br>

## 🟢 직관적인 작동 비유  
`git remote`는 스마트폰 연락처에 **친구의 이메일 주소를 등록하는 행위**와 같다.  
- 스마트폰에 주소록(Local)을 만들고, 거기에 원격지의 친구 이메일 주소(Remote URL)를 등록해 둔다.  
- 그래야 나중에 편지(코드 변경 사항)를 보낼 때 매번 긴 이메일 주소를 칠 필요 없이 친구의 연락처 별명(origin)만 클릭해서 바로 보낼 수 있게 된다.
