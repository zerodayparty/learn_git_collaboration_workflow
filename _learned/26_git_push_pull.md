# 🟩 git push, pull, fetch 명령어  

## 🟢 명령어 개요  

### 🟡 git push, pull, fetch란 무엇인가  
내 컴퓨터의 로컬 저장소와 인터넷 상의 원격 저장소(GitHub 등) 간에 저장된 소스코드 버전(커밋 기록)을 업로드하거나 다운로드하여 동기화하는 명령어들이다.  
- `push` = push (밀다)  
- `pull` = pull (당기다)  
- `fetch` = fetch (가져오다)  
- 해설: 협업을 진행할 때 내 수정 내용을 공용 서버에 반영하고, 다른 팀원이 올린 최신 코드를 내 컴퓨터로 업데이트하는 필수 과정이다.  

<br><br>

## 🟢 주요 사용법과 단어 풀네임  

| 명령어 | 명령어 영어 풀네임 및 단어 해석 | 해설 |  
| :--- | :--- | :--- |
| `git push [원격명] [브랜치명]` | `push` = push (밀다) | 내 로컬 컴퓨터의 특정 브랜치에 저장된 커밋들을 지정한 원격 저장소(예: origin)로 업로드한다. |  
| `git pull [원격명] [브랜치명]` | `pull` = pull (당기다) | 원격 저장소의 최신 커밋들을 내 로컬 컴퓨터로 다운로드함과 동시에, 현재 내가 있는 로컬 브랜치에 **자동으로 병합(Merge)**까지 시킨다. |  
| `git fetch [원격명]` | `fetch` = fetch (가져오다) | 원격 저장소에 새로운 커밋이 있는지 이력 정보만 내 컴퓨터로 가져오고, 내 코드 파일에 **합치지는(Merge) 않는다.** 안전하게 상태만 먼저 확인하고 싶을 때 사용한다. |  
| `git push -u origin main` | `-u` = set upstream (업스트림 설정) | 최초에 원격지와 로컬의 main 브랜치를 한 번 연결해 둔다. 이후부터는 매번 주소를 치지 않고 `git push`나 `git pull`만 입력해도 자동으로 origin의 main 브랜치와 통신하게 된다. |  

<br><br>

## 🟢 pull과 fetch의 치명적인 차이점  
초보자가 가장 많이 혼동하는 pull과 fetch의 차이는 **병합(Merge)의 유무**에 있다.  
- **git pull (가져와서 즉시 합치기)**: 원격 저장소에서 데이터를 다운로드하자마자 내 실물 파일과 합친다. 이 과정에서 내 수정사항과 겹치면 충돌(Conflict)이 바로 발생할 수 있다.  
- **git fetch (정보만 먼저 엿보기)**: 원격 저장소의 최신 소스코드를 다운로드는 하되, 내 작업 영역에 섞지 않고 별도의 임시 영역에 보관한다. 따라서 내 실물 파일은 전혀 변하지 않으며 안전하게 변경 사항을 미리 엿볼 수만 있다. 확인 후 안전하다고 판단되면 사용자가 수동으로 `git merge`를 실행해 결합한다.  
- **공식 공식**: `git pull` = `git fetch` + `git merge`  

<br><br>

## 🟢 직관적인 작동 비유  
- **git push**: 내가 스마트폰 메모장(Local)에 적은 메모 내용을 클라우드 서버(Remote)의 백업 폴더로 **'업로드(동기화)'**하는 행위이다.  
- **git pull**: 클라우드 서버(Remote)에 저장된 최신 공유 메모를 다운로드하여 내 폰의 메모장 앱 안에 바로 **'덮어쓰기 및 합치기'**하는 행위이다.  
- **git fetch**: 클라우드 서버에 새 메모가 올라왔다는 **'알림창(업데이트 내역)'**만 확인하고 내 폰 메모장은 아직 건드리지 않은 상태이다.
