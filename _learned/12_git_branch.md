# 🟩 git branch 명령어  
## 🟢 명령어 개요  
### 🟡 git branch란 무엇인가  
하나의 프로젝트 저장소 안에서 메인 개발 흐름을 해치지 않고, 독립적으로 다른 작업을 진행할 수 있도록 갈라져 나오는 새로운 개발 선(가지)을 의미한다.  
- `git` = Global Information Tracker (분산 버전 관리 시스템)  
- `branch` = branch (나뭇가지/가지)  
- 해설: 여러 사람이 동시에 서로 다른 기능을 개발할 때, 각자의 작업 영역을 분리하여 안전하게 코드를 작성할 수 있게 해준다.  

<br><br>

## 🟢 주요 사용법과 단어 풀네임  

| 명령어 | 명령어 영어 풀네임 및 단어 해석 | 해설 |
| :--- | :--- | :--- |
| `git branch` | `git` = Global Information Tracker<br>`branch` = branch (가지) | 현재 생성되어 있는 로컬 저장소의 브랜치 목록을 보여준다. (현재 머물고 있는 브랜치 앞에는 `*` 기호가 붙는다.) |
| `git branch [브랜치명]` | `branch` = branch (가지) | 입력한 이름으로 새로운 브랜치를 생성한다. (생성만 하고 그리로 이동하지는 않는다.) |
| `git switch [브랜치명]` | `switch` = switch (전환하다) | 지정한 이름의 브랜치로 현재 작업 공간을 이동한다. |
| `git switch -c [브랜치명]` | `switch` = switch (전환하다)<br>`-c` = create (생성하다) | 새로운 브랜치를 생성하고, 그 브랜치로 즉시 이동한다. |
| `git branch -d [브랜치명]` | `-d` = delete (삭제하다) | 작업이 끝난 대상 브랜치를 삭제한다. (단, 병합이 완료되지 않은 브랜치는 실수 방지를 위해 삭제가 경고와 함께 제한된다.) |
| `git branch -D [브랜치명]` | `-D` = Force Delete (강제 삭제) | 병합 여부와 상관없이 대상 브랜치를 강제로 삭제한다. |

<br><br>

## 🟢 직관적인 작동 비유  
`git branch`는 게임의 **세이브 슬롯**에 비유할 수 있다.  
- **메인 퀘스트 (main 브랜치)**: 게임을 정석대로 쭉 진행하고 있는 슬롯이다.  
- **서브 퀘스트용 세이브 (새 브랜치)**: 메인 스토리에 영향을 주지 않고, 보스 몬스터를 잡는 등 딴짓을 해보기 위해 세이브 파일을 복사해서 새로운 슬롯에 저장해 놓은 상태이다.  
- 슬롯을 복사하여 서브 퀘스트를 하다가 실패하더라도, 메인 퀘스트 슬롯(main 브랜치)에는 아무런 해가 가지 않는다.
