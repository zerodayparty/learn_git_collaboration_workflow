# 🟩 Git 로그 확인하기 (git log)  

지금까지 내가 만든 버전들의 기록(히스토리)을 확인하는 방법과 커밋 해시의 개념을 배운다. 기본 `git log` 사용법부터 상황에 맞는 여러 가지 유용한 옵션들을 학습한다.  

<br><br>

## 🟢 기본 로그 확인하기 (git log)  
- 저장소에 기록된 모든 버전의 상세 기록을 최신 순서대로 확인하는 명령어다.  
    ```bash
    git log  
    ```
- 출력 예시  
    ```text
    commit 76cst431351476ed7b9a2322e233e415207d52a4ta (HEAD -> main)  
    Author: boramson <boram@example.com>  
    Date:   Thu May 22 12:30:30 2026 +0900  

        feat: create test.txt  
    ```

<br><br>

## 🟢 커밋 해시란 무엇인가?  
- `git log` 명령어 입력 시 나오는 영어와 숫자가 길게 섞인 문자열을 말한다.  
    - (예: `76cst431351476ed7b9...`)  
- 깃이 버전을 구분하기 위해 부여한 고유한 ID 카드(주민등록번호)와 같다.  
- 나중에 특정 버전으로 돌아가고 싶거나 변경 사항을 비교할 때, 이 해시 값을 주소처럼 사용한다.  

<br><br>

## 🟢 로그 보기 좋은 옵션들  

### 🟡 커밋 기록을 한 줄로 보기 (--oneline)  
- 정보가 너무 많아 화면이 지저분할 때 사용하면, 커밋당 딱 한 줄로 깔끔하게 요약해 준다.  
    ```bash
    git log --oneline  
    ```
- 출력 예시  
    ```text
    76cst43 feat: create test.txt  
    3d5e2a1 initial commit  
    ```
    - 해시 값을 앞의 7자리(단축 해시)로 줄여서 보여준다.  

### 🟡 최근 기록만 제한해서 보기 (-n)  
- 최근에 만든 버전 몇 개만 빠르게 확인하고 싶을 때 사용한다.  
    ```bash
    git log -3  
    ```
    - 위 명령어는 최근 3개의 커밋 로그만 보여준다.  

### 🟡 🔥 브랜치 흐름을 그래프로 보기 (--graph)  
- 브랜치가 갈라지고 병합되는 흐름을 터미널에 선 모양(`*`, `\`, `/`) 그래프로 그려서 시각화해 보여준다.  
    ```bash
    git log --graph  
    ```

### 🟡 🔥 실무용 추천 조합 명령어  
- 전체 브랜치의 흐름을 한눈에 파악하기 위해 실무에서 가장 자주 쓰는 옵션 조합이다.  
    ```bash
    git log --oneline --graph --all  
    ```
    - `--all` 옵션은 현재 브랜치뿐만 아니라 모든 브랜치의 로그를 함께 보여준다.
